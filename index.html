Case Study: To‑Do List App

Problem

Users need a quick, lightweight way to manage daily tasks without creating an account or installing a heavy app.

Solution

I built a single‑page to‑do app using vanilla JavaScript. Tasks are stored in the browser's localStorage so they persist after refresh.

Challenges & Learnings

  • Managing state – keeping the UI in sync with the task list.
  • Handling edit and delete actions without reloading the page.
  • Learned about event delegation and DOM manipulation.

Future Improvements

  • Add categories and due dates.
  • Drag‑and‑drop reordering.
  • Dark mode toggle.

Case Study: Weather Dashboard

Problem

People want to check weather conditions quickly – current temperature, humidity, and a 5‑day forecast – all in one glance.

Solution

I connected to the OpenWeatherMap API using fetch() and async/await. Users can search by city name and see live weather data.

Challenges & Learnings

  • Handling API errors (e.g., city not found).
  • Displaying a loading spinner while data fetches.
  • Learned about CORS, API keys, and environment variables.

Future Improvements

  • Add geolocation support.
  • Show air quality and UV index.
  • Save favorite cities.
